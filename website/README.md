# Remorseless Havoc — Frontend

Static site. Canvas renderer, Sea-and-Spar-Between stanza generator
inlined in JavaScript, live state fetched from the Railway backend.

## Structure

```
website/
├── index.html                      # single HTML file, ~200 KB
├── public/
│   └── land_mask_3600x1800.bin     # binary landmask (3600×1800, 0.1°)
└── README.md
```

No build step, no dependencies beyond the bundled font imports.

## Pointing the frontend at a backend

Open `index.html` and edit `HAVOC_API_BASE` near the top of the main
`<script>` block:

```js
const HAVOC_API_BASE = (() => {
  const q = new URLSearchParams(location.search).get('api');
  if (q) return q.replace(/\/$/, '');
  return 'https://havoc.up.railway.app';   // ← your Railway URL here
})();
```

For quick local testing without rebuilding, append `?api=http://localhost:8000`
to the URL — the site picks that up automatically.

If `HAVOC_API_BASE` is empty, the site still renders with mock data
(no depletion, a handful of static vessels, no downloadable PDF).
That is the mode `file://` opens in by default.

## What each file is

- **`index.html`** contains everything: HTML, CSS, the canvas map
  renderer with Mip-Map pyramid, the full JavaScript port of
  *Sea and Spar Between*, and the fetchers that pull state from the
  backend. The stanza generator is inlined on purpose so it can be
  **read** from the page source: the appropriation of Montfort &
  Strickland's code is part of the work, not hidden behind a bundler.
  Key conceptual comments live at the hinges of the code:

  - Where the lattice is mapped onto the GFW grid (`gridToLattice`) —
    the literal cash-out of "as many verses as fish in the sea".
  - Where the depletion bitmap is consulted (`isDepleted`) —
    combinatorial abundance reduced to a bitmap that only ever goes
    from alive to gone.

- **`public/land_mask_3600x1800.bin`** is a flat binary: one byte per
  cell, 3 600 columns × 1 800 rows, row-major, south→north. 1 = land,
  0 = water. Fetched over HTTP at startup. Generated by
  [`../generation/generate_display_mask.py`](../generation/generate_display_mask.py).

## Catch-of-the-Day: size selector

The download panel exposes a dropdown with four size tiers — the full
archive stays the default (the scale of the day's erasure is part of
the work), the thinner cuts are navigation aids:

| Value | Rule | Approx. (for a 291 723-stanza day) |
|---|---|---|
| `full` | every catch | ~2 450 pp · ~51 MB |
| `digest` | every 10th catch | ~245 pp · ~5 MB |
| `excerpt` | every 100th catch | ~25 pp · ~500 KB |
| `vessel` | single top vessel, whole day | ~1 page |

The dropdown writes its value into `catchDownloadUrl()`, which appends
`?size=<value>` to the API call. The backend renders the non-full
sizes lazily on first request and caches them on disk
(`backend/app/pdf_builder.py`, `render_subset_on_demand`). Sampling is
uniform (every N-th catch), so the per-vessel poem structure and the
chronological order of catches are preserved.

## Deploying

### Cloudflare Pages

1. Point Cloudflare Pages at the repo, output directory `website/`,
   no build command.
2. Set `HAVOC_API_BASE` to your Railway URL in `index.html` before
   pushing (there is no env-var substitution — it's a static site).

### GitHub Pages

1. `git subtree push --prefix website origin gh-pages` (or a Pages
   action that publishes the `website/` directory).
2. Same edit: point `HAVOC_API_BASE` at the backend URL.

Both hosts serve `.bin` files with `Content-Type: application/octet-
stream` by default — no extra configuration needed.

## Running locally

```bash
python3 -m http.server 8090 --directory .
# http://localhost:8090/website/
```

Add `?api=http://localhost:8000` to hit a local Railway-image backend
(see [`../backend/README.md`](../backend/README.md)).
